**Frequency:** Every weekday (Monday - Friday)
**Time:** 9:00 AM EST
**Duration:** 15 minutes
**Location:** Microsoft Teams / Azure DevOps Boards

**Agenda:**

2. What will you work on today?
3. Are there any blockers?

**Participants:**
- Andrew Morton (Team Lead)
- Jayant Pathak (Developer)

- Round-robin updates (2-3 minutes per person)

## Sprint Planning

**Frequency:** First Monday of each sprint
**Time:** 2:00 PM - 4:00 PM EST
**Duration:** 2 hours
**Location:** Microsoft Teams + Azure DevOps Boards

**Agenda:**
1. **Sprint Goal Definition** (15 minutes)
   - Review Epic objectives
   - Define measurable sprint goal
   - Align with project roadmap

2. **Backlog Refinement** (30 minutes)
   - Review prioritized Issues
   - Clarify acceptance criteria
   - Identify dependencies
   - Estimate story points (if not already done)

3. **Capacity Planning** (15 minutes)
   - Team capacity for sprint
   - PTO and availability
   - Sustainable workload

4. **Task Breakdown** (45 minutes)
   - Break Issues into tasks
   - Assign work items
   - Identify technical approaches
   - Note any risks or unknowns

5. **Sprint Commitment** (15 minutes)
   - Finalize sprint backlog
   - Confirm sprint goal achievable
   - Set success metrics

**Participants:**
- Andrew Morton (Team Lead / Scrum Master)
- Jayant Pathak (Developer)
- Product Owner (when available)

**Deliverables:**
- Sprint goal documented in Azure DevOps
- Sprint backlog finalized
- Story points committed
- Initial task assignments

## Sprint Review

**Frequency:** Last Friday of each sprint
**Time:** 1:00 PM - 2:00 PM EST
**Duration:** 1 hour
**Location:** Microsoft Teams (recorded)

**Agenda:**
1. **Sprint Overview** (5 minutes)
   - Sprint goal reminder
   - Velocity achieved vs. planned
   - Key metrics

2. **Demo Completed Work** (40 minutes)
   - Demonstrate Done Issues
   - Show working features
   - Highlight technical achievements
   - Business value delivered

3. **Incomplete Items** (5 minutes)
   - Review items not completed
   - Explain blockers or challenges
   - Plan for next sprint

4. **Stakeholder Feedback** (10 minutes)
   - Q&A session
   - Collect feedback
   - Note change requests

**Participants:**
- Andrew Morton (Team Lead)
- Jayant Pathak (Developer)
- Stakeholders
- Product Owner
- Any interested parties

**Deliverables:**
- Demo recording
- Stakeholder feedback notes
- Updated Issues based on feedback

## Sprint Retrospective

**Frequency:** Last Friday of each sprint (after Sprint Review)
**Time:** 2:30 PM - 3:30 PM EST
**Duration:** 1 hour
**Location:** Microsoft Teams (team only)

**Agenda:**
1. **Set the Stage** (5 minutes)
   - Prime directive: Everyone did best job possible
   - Create safe environment
   - Confidentiality agreement

2. **Gather Data** (15 minutes)
   - What went well? (positives to continue)
   - What didn't go well? (challenges encountered)
   - What puzzles us? (things we don't understand)

3. **Generate Insights** (20 minutes)
   - Group similar items
   - Identify patterns
   - Root cause analysis
   - Vote on top priorities

4. **Decide What to Do** (15 minutes)
   - Select 1-3 actionable improvements
   - Define specific actions
   - Assign owners
   - Set measurable outcomes

5. **Close** (5 minutes)
   - Summarize decisions
   - Appreciate team contributions
   - Look forward to next sprint

**Participants:**
- Andrew Morton (Team Lead / Facilitator)
- Jayant Pathak (Developer)
- Team members only (no managers/stakeholders)

**Deliverables:**
- Retrospective action items (tracked as Issues)
- Process improvement notes
- Team health metrics

## Ceremony Best Practices

### General Guidelines
- ✅ **Start on time** - Respect everyone's schedule
- ✅ **Time-box discussions** - Stay focused and efficient
- ✅ **Action items tracked** - Every decision leads to Issue
- ✅ **Meetings recorded** - For team members who can't attend
- ✅ **Cameras on** - Increase engagement and connection
- ✅ **Prepare beforehand** - Review agenda and materials

### Communication Channels
- **Urgent:** Microsoft Teams chat
- **Blockers:** Tag in Azure DevOps + Teams mention
- **Updates:** Azure DevOps work item comments
- **Decisions:** Document in Azure DevOps wiki
- **Casual:** Teams channel messages

### Metrics Tracked
- **Velocity:** Story points completed per sprint
- **Cycle Time:** Time from To Do → Done
- **Lead Time:** Time from creation → Done
- **Throughput:** Number of Issues completed
- **Quality:** Bug rate, rework percentage
- **Team Happiness:** Retrospective sentiment
